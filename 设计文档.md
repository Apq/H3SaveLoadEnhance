# SaveLoadEnhance 设计文档

## 概述

插件在本次游戏进程内记住最后一次手动存档或成功读档的存档文件名。下次打开存档/读档界面时，自动选中该文件所在行并滚动到可见位置。

最后记录只存在内存中，不写配置文件；重启游戏后清空。插件启停由 HD Mod 插件列表控制。ini 只用于控制日志开关。

## 当前版本

- DLL 版本：0.5.2026.702
- 已验证游戏：Heroes3 2026.05.01（HD Mod）
- 最近实测日志：成功读档会记录文件名；取消/关闭读档窗口不会更新最后记录。

## 架构：5 个 Hook

| Hook | 地址 | 类型 | 职责 |
|------|------|------|------|
| `HH_SaveGame` | 0x4BEB60 | HiHook SPLICE_ THISCALL_ | 实际保存发生后记录手动存档文件名 |
| `HH_LoadSaveConfirm` | 0x58BFB0 | HiHook SPLICE_ FASTCALL_ | 读档确认流程开始时捕获当前选中文件名 |
| `Hook_AfterManualLoadGame` | 0x58C62E | LoHook | 手动读档路径中 `LoadGame` 返回后读取 EAX，成功才记录候选文件名 |
| `Hook_DialogShow` | 0x584EF4 | LoHook | 每次读档/存档对话框显示时自动选中记录文件 |
| `HH_DialogDestructor` | 0x583E10 | HiHook SPLICE_ THISCALL_ | 存档窗口关闭时建立短暂保存确认窗口；读档窗口关闭不记录 |

## 配置

配置文件固定读取 DLL 同目录下的 `SaveLoadEnhance.ini`。

```ini
[Logging]
DisableLog=0
```

`DisableLog` 默认值为 `0`：正常生成日志，并在插件加载时清理旧日志，保留最近 30 个。

当 `DisableLog != 0` 时，插件加载时不会初始化日志路径，不会创建日志文件，也不会扫描或清理旧日志；`WriteLog` 直接返回。

## 记录策略

### 手动存档

`HH_SaveGame @ 0x4BEB60` 包裹 `H3Main::SaveGame`。

签名：

```cpp
static int __stdcall HH_SaveGame(HiHook* h, DWORD thisPtr, const char* save_path, DWORD a3, DWORD a4, DWORD a5, DWORD a6)
```

必须透传全部 6 个原函数参数。存档界面可见时，或存档窗口刚关闭后的短窗口期内，`SaveGame` 真正被调用才记录文件名。这样避免用户只是打开、取消或关闭存档窗口时污染记录。

### 手动读档

读档使用两段确认，避免直接 Hook 底层 `H3Main::LoadGame @ 0x4BEFF0` 函数入口。该入口会被新游戏、战役、初始化、回退等非手动路径调用；实测在读档流程中包裹该函数会导致卡死。

1. `HH_LoadSaveConfirm @ 0x58BFB0`

   读档/存档对话框确认处理函数，对应反编译：

   ```cpp
   uint __fastcall FUN_0058bfb0(int param_1)
   ```

   进入原函数前，如果当前是读档界面，就从对话框对象读取当前选中文件名，写入 `g_pending_manual_load_name`，并设置 `g_pending_manual_load`。这里不直接记录，因为 `FUN_0058bfb0` 自身返回值不是底层 `LoadGame` 返回值。

2. `Hook_AfterManualLoadGame @ 0x58C62E`

   位于手动读档路径 `FUN_0058c560` 内部，紧跟 `call 0x4BEFF0` 之后：

   ```text
   0x58C629 call 0x4BEFF0
   0x58C62E mov ecx, [ebp-4]
   ```

   LoHook 在此读取 `HookContext::eax`。此时 EAX 仍是 `H3Main::LoadGame` 返回值。只有 `EAX != 0` 且存在候选文件名，才记录为“手动读档”；如果 `EAX == 0`，写失败日志并清理候选，不更新最后记录。

## 自动选中

`Hook_DialogShow @ 0x584EF4` 位于对话框显示/消息循环入口内部，消息循环开始之前。ESI 是对话框对象。

流程：

1. 通过 `[self+0x64]` / `[self+0x65]` 判断读档、存档或新游戏选图界面。
2. 如果是读档界面，清理旧的读档候选。
3. 如果是存档界面，设置 `g_save_dialog_visible` 并清理旧的存档关闭状态。
4. 如果已有最后记录，则在显示数组中按文件名查找目标索引。
5. 更新 `selected`、`topIndex` 和滚动条 tick。
6. 读档界面调用 `UpdateForSelectedScenario @ 0x5857D0` 和 `Redraw @ 0x584820` 刷新 UI。
7. 存档界面跳过刷新调用，只改索引和滚动位置，避免破坏文本输入框初始化。

## 对话框析构

`HH_DialogDestructor @ 0x583E10` 包裹对话框析构函数。

签名：

```cpp
static int __stdcall HH_DialogDestructor(HiHook* h, char* self, int delete_flag)
```

必须透传隐藏的 `delete_flag` 参数，否则栈清理量不匹配会崩溃。

- 存档界面关闭：不记录当前选择，只设置 `g_save_dialog_recently_closed`，等待后续 `SaveGame` 确认真正保存。
- 读档界面关闭：不记录，也不写普通关闭日志。取消/关闭读档窗口不能影响最后记录。

## 对话框对象关键字段

| 偏移 | 字段 | 说明 |
|------|------|------|
| +0x64 | 模式标志 | 1=读档 |
| +0x65 | 模式标志 | 1=存档 |
| +0x370 | topIndex | 列表顶部索引 |
| +0x374 | selected | 当前选中索引 |
| +0x1054 | 显示数组起始指针 | begin |
| +0x1058 | 显示数组结束指针 | end |
| +0x183C | 滚动条对象指针 | 列表滚动条 |

显示数组单项步长 `0xCA4`，文件名偏移 `0x33D`。可见行数：读档 18 行，存档 16 行。

## 日志

日志文件位于 DLL 同目录，命名为：

```text
SaveLoadEnhance_YYYYMMDD_HHMMSS.log
```

默认情况下，插件加载时读取日志开关、计算日志路径并清理旧日志，仅保留最近 30 个。

如果 `[Logging] DisableLog=1`，插件不创建日志文件，也不扫描或清理旧日志。

常见日志：

- `记录 '<file>'(手动存档)`：实际 SaveGame 调用确认后记录。
- `记录 '<file>'(手动读档)`：手动读档路径中 LoadGame 返回成功后记录。
- `更新 读档界面 ...` / `更新 存档界面 ...`：打开界面时自动选中了最后记录。
- `读档未确认:...`：确认路径未完成或 LoadGame 返回失败，不更新最后记录。

普通读档窗口关闭不再写日志，避免正常取消/关闭路径产生噪声。
